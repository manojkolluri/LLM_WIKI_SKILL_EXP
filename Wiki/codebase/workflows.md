# Workflows

Named runbooks for repeatable procedures in this codebase. Each workflow is a short, numbered procedure with a "files touched" list.

**Don't pre-write speculative workflows.** Add them here the first time you notice you're repeating a procedure across multiple changes.

## Workflow template

```
## How to <do the thing>

**When to use this:** <one-sentence trigger>

1. Step one — be specific.
2. Step two.
3. Step three.

**Files typically touched:** <comma-separated paths>

**Things to watch for:** <gotchas, side effects, cross-area updates>
```
